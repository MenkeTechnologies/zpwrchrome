zpwrchrome is a browser power-tool by MenkeTechnologies — built around the following pillars: a browserpass-compatible pass integration (with full-page CRUD manager + profile / credit-card autofill from profile/* and creditcard/* entries), a Chrono-inspired segmented download manager that takes over Chrome's default downloader, a JetBrains-style MRU tab switcher with scenes + opener-tree + minimap, a Tampermonkey-equivalent userscript host with a GM_* shim, a full-page screenshot capture that scrolls the active tab in viewport-sized steps and stitches the tiles into one PNG, a Wappalyzer-compatible technology detector running the vendored 3,993-fingerprint HTTPArchive corpus, a cyberpunk page-theme injector that paints arbitrary pages with the strykelang HUD palette, a Turn-off-the-lights cinema dimmer that overlays the page in near-black and spotlights any <video> element above it, a Reader mode that strips the page to its main article and renders it in the strykelang HUD with adjustable typography, a JSON viewer that auto-replaces plain JSON pages with a collapsible tree, a sibling XML viewer for application/xml / +xml vendor types plus .xml / .xsd / .xsl / .rss / .atom / .svg / .plist / .kml / .gpx / .opml / .fxml URLs, and a User-Agent switcher backed by chrome.declarativeNetRequest dynamic rules.

XML viewer

Auto-detects XML / SVG / RSS / Atom / plist / KML / GPX / OPML pages and replaces Chrome's raw <pre> dump with a DOMParser-driven collapsible tree in the strykelang HUD palette. No configuration page — sits silently on every tab and only acts when the content type matches. Skips the document when the JSON viewer already claimed it.

  • Detection is two-pass: a cheap looksLikeXml string pre-check (must start with <, must NOT be HTML or digit-led) gates the more expensive new DOMParser().parseFromString(raw, "application/xml") parse. If DOMParser surfaces a <parsererror> element, the viewer bails so the original payload + browser-level diagnostic stay visible.
  • Triggers on either: document.contentType matching application/xml / text/xml / <vendor>+xml, OR a path ending in any of .xml / .xsd / .xsl / .xslt / .rss / .atom / .svg / .plist / .kml / .gpx / .opml / .fxml, OR the page being just a <pre> whose body parses as XML (Chrome's default plain-text rendering).
  • Collapsible tree with separate color tokens for tag names (accent magenta #ff2a6d), attributes (orange #ff8c1a), attribute values (neon green #39ff14), text content (off-white), CDATA (magenta italic), comments (dim italic), and processing instructions (cyan italic). Leaf elements (<a>text</a>) stay on one line; mixed-content and nested elements get a collapsible child block with a <N children> preview when collapsed.
  • Toolbar at top: live filter (Ctrl/Cmd+F focuses it; matches tag names / attributes / values / text / CDATA / comments and auto-expands ancestor collapsibles to reveal hits), copy (raw source), pretty copy (re-indented via lib/xml-format.js's single-pass leaf-compacting formatter), minify copy (inter-tag whitespace stripped, CDATA / comments verbatim), expand all, collapse all, and raw (toggle between the tree and the original bytes).
  • Per-node ⧉ xpath chip on hover — copies a 1-based XPath (/feed[1]/entry[3]/title[1]) of the addressed node to the clipboard. navigator.clipboard.writeText is used when available; falls back to the legacy hidden-textarea + execCommand trick so the button still works on plain-http (non-secure-context) pages.
  • HTTP(S) URLs in both text content and attribute values are auto-linkified so feed / sitemap / RSS payloads navigate without copy-paste.
  • Pure-helpers contract: lib/xml-format.js exports looksLikeXml, isXmlContentType, hasXmlExtension, prettyPrint, minify, countTags, xmlByteSize, formatXPath. 25 unit tests pin auto-detect, formatting, and XPath shape (1-based predicates, defaults to [1] when index missing).

Post-download commands

Run a per-rule argv command the moment a download flips active → done. Rules live at Post-download Commands in the sidebar and match the finished file's basename against a glob (*.zip, *.tar.gz, case-insensitive). Matching is first-rule-wins — reorder with the up/down arrows.

  • Argv-style only. The command is parsed shlex-style on the JS side and shipped as an array to the host's run.spawn action. The host calls std::process::Command::new(argv[0]).args(argv[1..]) — no shell, no system(3), no quoting surface. A {path} containing whitespace or shell metacharacters stays one argv entry.
  • Placeholders: {path} (full path), {dir}, {name} (basename), {base} (basename without extension), {ext} (no dot).
  • Confirm flag: per-rule. When set, finishes a download → Chrome notification with Run / Skip buttons. The pending argv survives SW suspends via chrome.storage.session so the buttons still work after the worker idles out.
  • Output: stdout / stderr captured (64 KiB cap each), exit code reported, 30s default timeout (max 5 min) kills runaways with code 124. Result surfaces as a notification.
  • Shell features (pipes, redirects, &&) require wrapping the rule's command in bash -c '…' explicitly — opting in keeps the default surface injection-free.

Reader mode

Strips the active page to its main article and renders it inside a fixed-position overlay with the strykelang HUD palette. No Readability.js dependency — purely heuristic extraction. Trigger via the reader-mode command, the toolbar context menu — Reader mode (this tab) — or the global nav strip on any settings page.

  • Three-step extraction: largest <article> → <main> / [role="main"] → densest paragraph cluster outside nav / aside / footer / ad / share / comment / widget containers.
  • Clone is sanitized before render: <script> / <style> / <iframe> / <form> / <nav> / inline on* event handlers all stripped. Relative URLs resolved to absolute so images and links still work inside the overlay.
  • Four themes (cyberpunk default, classic-dark, classic-light, sepia) × three font families (mono, serif, sans). Font-size / line-width / line-height sliders + optional CRT scanlines. Settings UI at scripts-manager/reader-mode.html with a live preview pane.
  • Top-bar live controls: A− / A+ bump font size on the fly and persist; Esc or the × closes. Original page DOM is untouched the whole time — the overlay just sits on top.

Turn off the lights (cinema mode)

Port of the Turn Off the Lights Chrome extension. Trigger via the lights-off command (bind at chrome://extensions/shortcuts), the toolbar context menu — Turn off the lights (this tab), or the global nav strip on any settings page — which opens the lights-off settings UI.

  • modal/lights-off.js walks every visible <video> element, walks the ancestor chain up to <body>, and sets z-index: 2147483647 + (if needed) position: relative on every node so the player escapes any overflow: hidden or stacking-context clip.
  • A full-viewport <div> overlay sits one z-index below the video chain at 2147483646 with the configured opacity, color, and CSS fade. Click the overlay or press Esc to undim — both restore the original z-index / position values via the data-zpwr-lights-lifted attribute we tagged on each touched element.
  • Settings (scripts-manager/lights-off.html): opacity 0–1 slider, fade duration ms, overlay color picker, auto-on toggle (dim automatically on any page that has a <video>), blocklist / allowlist textarea (one host per line, dot-anchored subdomain match), live preview pane.
  • Per-host routing matches the cyber-theme injector model: all and blocklist treat the textarea as exclusions, allowlist flips polarity. Settings persist in chrome.storage.local under the lights.off key.

Cyberpunk page-theme injector

modal/cyber-theme.js runs at document_start on every http(s) tab and paints the page with the strykelang HUD palette. Configure at Open theme injector on the toolbar context menu (or popup → theme ▸). Knobs:

  • intensity — subtle = links + headings + scrollbars only / medium = + body bg + form fields + code blocks / full = + tables, cards, dimmed images.
  • dark mode — smart overlay using color-scheme: dark + targeted overrides for common white-card patterns: Amazon AUI .a-box / .order-header / .delivery-box / .bia-content, generic [class*="card|panel|widget"], ARIA dialogs, and inline background: white|#fff|rgb(255,…). Deliberately not filter: invert() — that approach inverted already-dark pages too.
  • forceMono — Share Tech Mono everywhere except icon-font carriers (<i> / <svg> / [class*="icon|fa-|material-icons|material-symbols|lucide|phosphor|glyphicon"] / [data-icon|data-lucide|data-cds="Icon"|data-radix-icon]) so Anthropicons, Material Symbols, Lucide, etc. keep their glyphs.
  • scanlines — CRT-style horizontal overlay via body::after.
  • blocklist / allowlist — per-host textarea (one host per line). Settings broadcast to every open tab via chrome.storage.onChanged; no reload needed.

Full-page screenshot

Bind screenshot-full-page at chrome://extensions/shortcuts (or right-click the toolbar icon → Full-page screenshot (this tab)). The capture:

  • Scrolls the active tab through its full scrollHeight in viewport-sized steps with a 200 px overlap, captures each viewport via chrome.tabs.captureVisibleTab, and stitches the tiles on an OffscreenCanvas in the service worker.
  • Sleeps 600 ms between captures (Chrome enforces MAX_CAPTURE_VISIBLE_TAB_CALLS_PER_SECOND ≈ 2); retries with exponential backoff (1.1 s → 2.5 s → 5 s) on quota errors.
  • Pins fixed / sticky elements to position: static !important during the capture so they don't appear N times in the stitched output; original styles are reverted on completion.
  • Streams the PNG to the host in 512 KiB base64 chunks via dl.writeFileChunk (Chrome's host → ext native-messaging cap is 1 MiB), then renames the upload .part file to the chosen filename in your configured downloads directory.
  • Hard caps: 60 tiles max (runaway infinite-scroll guard) and ~16k × 16k output pixels (memory bound). No chrome.debugger permission required, no permanent yellow banner.

The download-manager surface (this Settings UI, the queue manager, Rule System, Extension Filter) is a homage to Chrono Download Manager — reimplemented for Manifest V3 with a native Rust segmented downloader (zpwrchrome-host) that handles HTTP Range requests, cookie / user-agent forwarding, detached worker processes, and a file-state model surviving SW restarts.
