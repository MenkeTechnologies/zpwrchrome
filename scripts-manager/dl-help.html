Full-page screenshot

Captures the active tab edge-to-edge — even the parts off-screen. Trigger via the toolbar context menu (right-click the icon → Full-page screenshot (this tab)) or bind screenshot-full-page to a key at chrome://extensions/shortcuts.

What it does:

  • Scrolls the page in viewport-sized steps with a 200 px overlap so sticky / fixed elements stitch cleanly.
  • Captures each viewport via chrome.tabs.captureVisibleTab; sleeps 600 ms between captures (Chrome's hard quota is ~2 captures / s). Quota errors retry with exponential backoff.
  • Temporarily forces every position: fixed / sticky element to static during capture, restoring originals on completion.
  • Stitches tiles on an OffscreenCanvas in the service worker, writes a PNG to your configured downloads directory via the host's chunked dl.writeFileChunk protocol (works around Chrome's 1 MiB host → ext native-messaging cap).
  • Hard caps: 60 tiles, ~16k × 16k output pixels — prevents memory blow-up on runaway infinite-scroll pages.

No chrome.debugger permission is required — none of the leading consumer screenshot extensions use it either, because it triggers a permanent yellow "DevTools attached" banner.

Batch Descriptors

Used to expand a URL pattern into multiple downloads. Defined as [start:end] or [start:end:step].

Examples:

  • https://site.com/img[1:100].jpg → 100 URLs, img1.jpg through img100.jpg
  • https://site.com/img[01:99].jpg → 99 URLs, padded (img01.jpg through img99.jpg)
  • https://site.com/p[0:200:10].html → 21 URLs, stepping by 10
  • https://site.com/[a:f].txt → a.txt, b.txt, …, f.txt

Naming Masks

A naming mask is a template that produces the saved filename. Tokens are *name*, *ext*, *url*, *host*, *date*, *time*, *size*, and *subdirs*. Tokens not in square brackets are required.

  • *name*.*ext* → keep original name + extension (default)
  • *host*/*name*.*ext* → group by source host
  • *date*-*name*.*ext* → prepend YYYY-MM-DD
  • *flat* → like *url* but slashes become underscores

File names can include sub-folders (dir1/dir2/*name*.*ext*).

Rule System

Rules combine conditions over a job's metadata with an optional naming mask. A condition is a boolean expression with star-bracketed identifiers as variables and square-bracketed identifiers as constants.

Condition syntax:

  • *field* — variable from a job (*name*, *ext*, *url*, *host*, *state*, *size*, *time_add*, *time_done*)
  • [constant] — pre-defined constant ([finished], [interrupted], [3_days_ago], [ext_image], …)
  • {r_name} — reference another rule by internal name
  • == != < > <= >= — comparison
  • && || ! — logical
  • *ext*.is([ext_image]) — extension-filter membership

Useful functions inside […]:

  • [kb(N)], [mb(N)], [gb(N)] — size constants in bytes
  • [N_days_ago], [N_hours_ago] — timestamps
  • [regex("…")] — compile a regular expression
