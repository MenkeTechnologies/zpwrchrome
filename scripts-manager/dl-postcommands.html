Post-download commands

Run a custom command when a finished download matches a glob. Rules are matched top-to-bottom, first-match-wins — reorder with the ↑ / ↓ arrows.

Commands are parsed argv-style (shlex). The available substitution placeholders are {path} (full path), {dir} (directory), {name} (basename), {base} (basename without extension), and {ext} (extension, no dot). A {path} containing spaces still passes as a single argv entry — there is no shell, no injection surface. Need pipes or redirects? Wrap your command in bash -c '…' explicitly.

On Confirm Name Glob Command Reorder

How matching works

  • Glob is matched against the file's basename, case-insensitive. * matches any chars, ? matches exactly one. *.tar.gz matches archive.tar.gz; *.zip matches foo.ZIP.
  • First-match-wins: a download matching both *.zip and * fires only the rule that appears first in the list.
  • Disabled rules are skipped entirely (the next-down rule is considered).
  • Confirm shows a Chrome notification with Run / Skip buttons before spawning. Useful for commands you want to opt into per download.
