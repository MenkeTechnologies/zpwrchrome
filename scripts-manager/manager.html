quick start

  1. Open chrome://extensions and turn on Developer mode (top-right toggle).
  2. Click ＋ on this dashboard to create a new userscript.
  3. Paste a ==UserScript== block with at least @name and @match.
  4. Click save. The script registers immediately on every page that matches.

supported metadata directives

  • @name @namespace @version @description @author @icon
  • @match @include @exclude
  • @run-at · values: document-start, document-end, document-idle
  • @grant · listed for reference; the shim exposes the standard GM.* API regardless
  • @require @resource · parsed, not yet loaded (Phase 2)

GM.* API exposed

  • GM.getValue(key, fallback) → Promise
  • GM.setValue(key, value) → Promise
  • GM.deleteValue(key) · GM.listValues()
  • GM.setClipboard(text)
  • GM.openInTab(url, opts)
  • GM.addStyle(css) · GM.addElement(parent?, tag, attrs)
  • GM.notification({ text, title })
  • unsafeWindow alias to window (USER_SCRIPT world)

limits (vs Tampermonkey)

  • No GM_xmlhttpRequest / cross-origin proxy yet (Phase 2).
  • No @require external library fetch (Phase 2).
  • No @resource blob fetch (Phase 2).
  • No sync GM_ cache (all GM.* return Promises).
  • Chrome's match-pattern spec applies — wildcards in path, scheme is one of *|http|https|file|ftp|urn.
