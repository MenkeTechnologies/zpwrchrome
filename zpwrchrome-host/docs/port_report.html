+EXTENSIONS (NOT IN UPSTREAM)

Three additive modules live under src/extensions/ — features browserpass-native does not have. browserpass-extension never sends these action names, so wire compat with upstream is preserved. Each is dispatched by bin/zpwrchrome_host.rs before falling through to the ported BP dispatcher.

ModuleRust LOCAction(s)Purpose
src/extensions/otp.rs120otpShells pass otp <entry> with the matching store's PASSWORD_STORE_DIR. Upstream v3 dropped OTP; we add it back because zpwrchrome's PASS popup needs it.
src/extensions/search.rs175searchHost-side fuzzy + substring scoring across every configured store. Faster than client-side scoring for large stores. Pure scorer pinned by tests/extensions_search.rs.
src/extensions/dl.rs793dl.add / dl.list / dl.pause / dl.resume / dl.cancelFile-backed segmented HTTP/HTTPS downloader. dl.add detaches a worker (zpwrchrome-host --dl-worker <gid>); state persists at $XDG_CACHE_HOME/zpwrchrome/dl/gid_NNNNNN.json. HEAD probe → N parallel Range GETs → atomic state-file rewrites; pause/cancel via state-file flags polled by the worker between chunks. Retry 200ms × 3ⁿ on transient errors. End-to-end verified by tests/extensions_dl_integration.rs against a Range-aware HTTP fixture.
